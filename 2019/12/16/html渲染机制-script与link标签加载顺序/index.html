HTML渲染机制-script与link标签加载顺序

Posted on 2019-12-16 | In Web | | Visitors:

浏览器在解析HTML的过程中可能会对DOM的渲染或者其他资源的加载造成阻塞，实际上这是浏览器内核出于对各种标签里面的可能执行内容，对浏览器渲染效率的考虑而做出的结果。

我做了一个简单的测试，用setTimeout函数模拟网络延迟，观察console界面输出顺序和network的加载情况

script标签和link css标签

1.css


浏览器版本：chrome 79.0.3945.88
结果显示无论script和css在head里的顺序如何，两个请求同时发起，虽然没有阻塞其他资源的请求，但是script标签会阻塞DOM的渲染，所以一般情况下，script标签写在body的底部。
PS: 有些地方会把script写在body标签之后，html标签之前，其实这样是不符合标准的，html只有两个标签，head和script，但是由于大多数浏览器都有容错（实现不一定一样），对于chrome来说，head和body之间的标签会被合进head，body和html之间的标签会被合进body。

link icon标签


浏览器版本：chrome 79.0.3945.88
结果证明，浏览器同时请求js和css，请求完成之后执行js内容，最后才请求icon文件。

js的执行

当我们把js的延迟设为0，css的延迟不变的时候，会发现当js标签放在css标签后面的时候，会等待css加载完之后再执行；当js标签放在css标签之前的时候，js就会在请求返回之后立即执行。这是因为考虑到js可能会对css样式进行操作。

结论

  1. 浏览器并行加载,顺序执行,icon标签最后加载
  2. js加载阻塞DOM的渲染
  3. css加载阻塞后面js的执行
